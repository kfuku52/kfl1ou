# Security policy

## Supported versions

Security fixes are applied to the current 2.4.x release line and to `main`.
Older releases may receive a fix only when the same change can be applied
safely.

## Reporting a vulnerability

Do not open a public issue for an unpatched vulnerability. Use
[GitHub's private vulnerability reporting](https://github.com/kfuku52/kfl1ou/security/advisories/new)
or email the maintainer at `kenji.fukushima@nig.ac.jp`.

Include the affected version, platform, reproduction steps, expected impact,
and any suggested mitigation. You should receive an acknowledgement within
seven days. The maintainer will coordinate validation, a fix, and disclosure;
timing depends on severity and release complexity.
